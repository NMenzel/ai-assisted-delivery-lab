# Reviewer Decision Tree

## Approve

Approve only when requirements are met, risk is understood, tests are adequate, accessibility and security considerations are addressed, and no blocking architecture concerns remain.

## Request Changes

Request changes when correctness, security, accessibility, maintainability, test coverage, or hidden behavior changes are unresolved.

## Escalate

Escalate when the change affects authentication, authorization, data privacy, compliance claims, production incidents, destructive actions, or accessibility conformance statements.
