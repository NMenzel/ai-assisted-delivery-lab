# AI Code Smells

- Overly broad abstractions for one use case.
- Hallucinated framework APIs.
- Excessive defensive code without clear failure model.
- Silent behavior changes during refactor.
- Weak tests that only assert rendering.
- Comments that restate code instead of explaining decisions.
- New dependencies for trivial functionality.
- Accessibility attributes used to compensate for poor semantics.
- Security checks described in comments but not enforced.
